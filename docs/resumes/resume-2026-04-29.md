## Launching the eval server

Postern provides the eval server. The Pharo image is disposable —
all source lives in `src/` (Tonel format).

```bash
# from repo root
make setup     # download Pharo 12 + load all Tonel packages (first time)
make start     # launch Pharo GUI with eval server on :${EVAL_PORT:-8422}
make status    # health check — confirms eval server + package count
make stop      # kill the VM (no save — image is disposable)
make rebuild   # destroy + recreate from scratch (proves source is complete)
```

`make rebuild` MUST always work. If it does not, the Tonel source
is incomplete — fix the source, never the image.

For a second concurrent image (parallel agent dispatch):

```bash
EVAL_PORT=8423 make start
```

Each port gets its own `.pharo-<port>.pid` and `.pharo-<port>.log`.
`make start` probes the target port before launching and fails if
another eval server is already responding.

## Running tests

```bash
make test       # full Messaging suite via ClaudeMessagingTestSuite
make test-fast  # excludes slow + integration tests
make test-full  # includes live-API integration (requires ANTHROPIC_API_KEY)
make lint       # Renraku via ReCriticEngine — zero non-clean lines required
```

`make lint` from the Bash tool is the canonical lint gate. Per-method
`m critiques` misses class-level rules and is a debugging tool only.
Pipe through `grep -v ': clean$' | grep -v '^$'` to see only
findings.

## Dispatching kwb missions

All Smalltalk implementation work is delegated to the kwb agent via
ethos mission contracts. The COO does not write code.

```bash
# 1. Branch first (Iceberg commits to current branch)
git checkout -b <prefix>/<bead-id>-short-description main

# 2. Write contract to .tmp/missions/<bead>.yaml
# 3. Dispatch
ethos mission dispatch --worker kwb --evaluator <djb|bwk|adb> \
  --write-set <paths> --criteria <criteria> \
  --file .tmp/missions/<bead>.yaml

# 4. Spawn the agent (Agent tool, run_in_background=true)
#    Prompt: "Read your mission contract: ethos mission show <id>.
#             Execute it. Submit: ethos mission result <id> --file <path>."

# 5. After kwb reports back: cross-check make lint from CLI,
#    read the diff end-to-end, ethos mission close <id> or reflect+advance.
```

**Spec review is required before every dispatch.** Run
`feature-dev:code-architect` on the spec; only APPROVED specs go to
the worker. Skipping spec review costs rounds.

**Evaluator picks**:
- `djb` for security work (auth, TLS, API-key handling)
- `bwk` for wire-format / Go SDK parity verification
- `adb` for infra / CI / cross-repo tooling
- (Smalltalk self-review by kwb is not allowed.)

See CLAUDE.md "Delegation with Missions" for the full contract
discipline.

## Workflow per change

1. Pick a bead with `bd list --status=open`.
2. `bd update <id> --status=in_progress`.
3. Branch from main.
4. Dispatch kwb mission with spec-reviewed contract.
5. After kwb reports, run `make lint` and `make test` from CLI.
6. Update CHANGELOG / ROADMAP / DESIGN as needed.
7. Open PR, address review (Copilot + Bugbot), merge squash.
8. Tag if the bead crosses a version boundary.
9. `bd close <id>`.
